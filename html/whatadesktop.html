What Is It?

WhataDesktop is my solution to easily finding new desktop wallpapers.  I’ve always hated the stock wallpapers and I never enjoyed actively finding new ones.  Most the sites I went to were difficult to use or required too much user input  -- I’m not very creative I can only think of so many search words.  Instead, I decided to build a website that brought wallpapers to me.

WhataDesktop is is an AngularJS application that displays images collected by a python script.  The application is displayed in such a way that I can preview how a wallpaper will appear on my computer before I download it.  I can use specific hotkeys to easily navigate through the image queue, remove or save images, and download images.  

Software

There are two components to this website: a front end application that uses a REST API and a python script that uses Reddit to source images.  Once a day (schedule via cron) my python script will execute and collect any imgur links on the front page of chosen subreddits (e.g. wallpapers), it will store those links in a database with associated metadata.  These images are queried from the client to create image ‘queues.’

Image Collection

There is nothing special here.  This is my least favorite part about this application and it’s something I’ve been meaning to rework.   Like most python scripts I write, they exist solely to accomplish one task. As such, the python code tends to be, uh, unprofessional.  I’m not a python developer and I don’t know python coding standards -- this is all blatantly obvious.

The python script I use parses a reddit json url and searches for imgur links.  When an imgur link is found it’s metadata is retrieved, parsed, and stored in a database. The database contains the asset URL, imgur identifier, width, height, and a unique hash that identifies the image and it’s dimensions.  

API

My API queries the database for various images and returns a list of images back to the client.  Each image contains specific metadata like aspect ratio, width, and size; it also contains a set of URLs that point to different image resolutions.  

Show JSON response

There are predefined filters that are used to sort images according to popularity and newness; the default filter is to randomize the list of images. An images popularity is determined by the number of people that liked or disliked an image; the more an image gets saved the more it gains in popularity. A more traditional filter also exists for targeting specific width or height requirements and aspect ratios.

AngularJS Application

This website was my first exposure to angular JS, so I spent most of my time on this part of the application. It uses the concept of an ‘image queue’ that is traversed to display images that auto resize to fit your screen.  It also uses ngroute to separate logical areas of the application (e.g. queue/random, saved/, and deleted/), this was also new to me and I’m glad I did it because it has all the benefits of a single page application without the nasty single URL.

In my opinion the most interesting part of the application is the image queue.  It is a data structure that acts a carousel - circular queue - that can be traversed forward and backward.  It also has the ability to add, remove, and ‘seek to’ images in the queue.  Essentially, an image queue is a dynamic array backed by a dictionary, this gives us:

  • O(n) delete and seek
  • O(1) contains, get, add, and update

After writing these, I realized I can get constant delete and seek by changing the dynamic array to a linked list and keeping a  reference to an images node in the dictionary.  Oh well, something to do at a later date.
